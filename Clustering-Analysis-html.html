"""1.2) Using only x and y (exclude org_cluster column), use the k-means algorithm to
cluster the dataset number of clusters from 1 to 10. If you are using sklearn, set a
fixed random state to 0. Plot inertia (within-cluster sum of squares) against the
number of clusters. What is the best number of clusters for this data and why? """

# Extracting x and y columns (excluding org_cluster)
x = df1[['x', 'y']].values

# Compute k-means clustering for k = 1 to 10 and store inertia values
inertia = []
for k in range(1, 11):
    kmeans = KMeans(n_clusters=k, random_state=0)
    kmeans.fit(x)
    inertia.append(kmeans.inertia_)

# Plot inertia vs. number of clusters
plt.figure(figsize=(8, 5))
plt.plot(range(1, 11), inertia, marker='o', linestyle='--')
plt.xlabel('Number of Clusters (k)')
plt.ylabel('Inertia (Within-Cluster Sum of Squares)')
plt.title('Elbow Method for Optimal k')
plt.xticks(range(1, 11))
plt.grid(alpha=0.3)
plt.show()

Inertia measures the within-cluster sum of squares. A lower inertia indicates that data points are closer to their cluster centres, suggesting better clustering. The optimal number of clusters is 3, based on the "Elbow Method". The elbow point is identified at k=3, where the rate of decrease in inertia slows significantly, indicating a balance between simplicity and representation of the dataset's structure.

Adding an extra cluster at k = 4 increases model complexity without providing a substantial improvement in within-cluster cohesion. Increasing the number of clusters makes the model harder to interpret. At k = 4, the clusters may start to capture more noise or minor patterns that don't significantly contribute to understanding the dataset. At k=3, adding more clusters yields diminishing returns in reducing inertia
